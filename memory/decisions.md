# 设计决策与约束（改首页/导航/搜索前必读）

## 🚫 红线（用户明确反对，勿再提/做）
- ❌ 自动检测输入文字系统 → 同时查所有库 → 展示一长页（体验极差，已试过失败，输一字就刷屏）
- ❌ 通用自动识别脚本类型（Latin 字母无法区分英/法/德/西）
- ❌ 首页纯空白搜索框（新站伤新用户认知 + 首页 SEO 关键词）

### 版权红线（2026-06-06，AdSense 商业化前定。曾误踩，已止血）
- ❌ **对外文案永不点名具体版权辞典作为数据来源**。persons 67万条实源自《世界人名翻译大辞典》（商务印书馆，有版权）；公开承认=主动递侵权证据+踩 AdSense"未授权版权内容"红线。about 页曾写"包括《世界人名翻译大辞典》词条"，已删（commit 083357e）。
  - ✅ 可写：遵循公开译名**规范/规则**（新华社译名标准、外语译音表）——方法论，正面，不构成版权问题。naming-rules/gov-titles 满屏"参照新华社手册"OK，那是声明依据的标准，非复制词条。
  - 区别关键：「遵循XX标准」（方法论，可）vs「数据=XX辞典词条」（承认复制，禁）。
- ❌ **永不加"导出全部/批量下载"入口**。站定位=单条查询工具（用户输入才返回单条），法律上属"事实性查询"，比"整本汇编公开陈列/分发"风险低一个量级。批量导出会把性质变成分发汇编=送上门。
- 守住这两条线即可商业化（上 AdSense）；不必为版权重建 67万数据（成本高、收益低）。单条译名是客观读音对应=事实，不受版权；受保护的是整本的选择+编排，而我们不对外暴露该结构。
- ⚠️ **2026-06-18 补漏**：`data/transliteration/*.json` 的 `raw_note` 含「《世界人名翻译大辞典》/新华社/商务印书馆/ISBN」，曾被渲染到 /convert「参考来源」。已修：客户端两过滤器 + 服务端 /api/lang-table `stripCitations`（`CITATION_RE` 关键词剥离）双层拦截，辞典名既不显示也不外发。**以后改 /convert 或 lang-table API 勿移除该过滤。**

### 程序化页剪枝（2026-06-18 定，勿重提）
- 俄名人页 /ru/name 共 1927，单人页占 93%（模板薄）。**但不剪**：`topSitelinks` 下限 ≥20（每页人物都在 20+ 语言维基有词条，建库时已按知名度筛过），无垃圾尾巴可剪，剪只会误杀普京/列宁等合法页。模板薄≠低质——每页是真实知名人物参考资料。AdSense 靠编辑型内容（总则/4专题/36语种/辞典）撑质量，不靠删页。

## 已定论

- /convert 已有彩色分段展示（汉字+字母颜色连线），可复用

## 信息架构（2026-06-02 重构定论）
**用户场景收敛为 3 主任务：**
1. 查专名既定译名（人物+地名，外→中）= 极高频 = 首页主搜
2. 普通名音译（选语言，外→中）= 中频 = 二级 /convert
3. 中文名→外文罗马化（拼音/粤拼/威妥玛）= 中频 = **功能未建**

**核心二分（搜索逻辑）：**
- A 类·专名库（名人库俄韩日 + 地名 places）：集合小结果少 → 进主搜实时联想，分「人物/地名」区
- B 类·音译辞典（persons 67万）：海量，前缀匹配刷屏 → 隔离第二步选语言精确查
- ⚠️ 刷屏根因 = `${q}%` 前缀匹配在67万表；persons 实为姓氏/教名词条库，限量匹配也可进主搜。/en 已限 8/5 缓解

**导航原则：按任务，不按语言/数据源**
- /ru /ko /ja /places 是"数据源当入口"的错误切分
- 导航 5 项：人名地名翻译(=首页) / 中文名转外文 / 外文音译 / 命名规则 / 机构职位(/gov-titles)
- 语言收进搜索结果做分组标签，永不进导航
- ⚠️ 移动端 5 项可能挤，待移动端适配统一处理
- ⚠️ 当前 NavBar「中文名转外文」暂指 /naming-rules/sinosphere 占位，真罗马化引擎(场景3)建好后换链接

## /ru 逐词直译（2026-06-05 拍板，用户定）
- **呈现 = 直给，不做联想**：输入按空格切词，每词查中文行内直给。例：`Владимир Путин` → `Владимир 弗拉基米尔  Путин 普京`。**不出**小名/父名联想列表、不做性别筛选（推翻旧 /ru 嵌套卡片+联想）。
- 查词优先级：① `lib/ruDict.ts` 静态字典（data/russian/given_names.json，教名/父名/爱称，准）→ ② russian_names 现表精确兜底（姓氏/名人如 Путин，**只读不写**）→ ③ 俄语音译引擎兜底（lib/transliterate `俄語` 表，标"推测"灰色）。
- 反查（中→俄）：仅字典+DB 命中；姓氏中文反查可能 miss（如 普京），可接受。
- API `/api/ru-translate`；旧 `/api/search-ru` + russian_names 嵌套搜索已弃用（未删，无引用）。
- ⚠️ russian_names 现表 = 同源 tingroom(itemid=105) 旧抓 493(残缺) + wikidata，混装；本次没动它，只读兜底。

## /convert 音译表 URL 预选（2026-06-05）
- 支持 `?lang=<langFile>&q=<text>`（可选 `&country=`）：从语言页跳来直接预选对应国家+语言并出结果，免重选。顶部提示「已为你打开「X」音译表·下方可切换其他语言」，国家 pill 全列表=所有语言入口。
- langFile key 见 data/country_languages.json（如俄=`俄語`）。⚠️ **只有拉丁/西里尔类语言有音译表**；`朝鮮語`/`日語` **无表**（韩日靠汉字非音译）。故 /ru CTA 带 `lang=俄語`；/ko 无表→CTA 改指 `/naming-rules/korean`（非音译）；/places 多语言→通用不预选。
