## 待改进清单（按优先级）

### P0 数据可信度（伤译者专业信任，最高优先）
1. **/convert 德语乱码**：Müller→"？-{于}-？？埃？"。变音符(ü/ö/ä)+双字母(ll)规范化坏。德/法是高频语种，崩了很伤。需查 normalizeForLookup + 德語表。
2. **/en 同名多译排序**：Smith→斯米特先于史密斯。同一拼写多译名时，约定俗成/英语读法应优先。无频率数据→可按 nationality 启发式（拉丁常见名优先英美读法）。
3. **/places ASCII↔变音符**：Tokyo 打不中 Tōkyō。译者打 ASCII，必须去变音符匹配。+ 过滤"见X"交叉引用词条。
4. **/en 数据噪声**：Lyon→"男子教名,亦作姓用"。过滤 chinese 为纯释义/注释的 persons 词条（todo 已记）。
5. **/pinyin 高频字缺失**：wei 无"伟"。字库/频率表漏核心常用字，张伟查不到很扎眼。

### P1 体验一致
6. **/ko 罗马字搜索**：Seojun→空但 placeholder 承诺。要么修 english 字段模糊匹配，要么改 placeholder 去掉罗马字承诺。
7. **/convert 法语等纯音译质量**：Dupont→迪蓬？。纯音译对有约定俗成的名（杜邦）必错。引导文案应更明确"仅用于无通用译名的普通名"。

## ✅ 修复记录（2026-06-05 全部上线验证）
1. **德语音译乱码** → 修：lib/transliterate 加 DE_CONSONANTS 手工列序(绕过被抓乱的列头)、处理"元音"纯辅音行、cleanChinese 清 `-{}-` 变体标记、德语规范化(双辅音收一/ck→k/ß→s)。Müller→米莱尔、Becker→贝克尔、Hoffmann→霍夫曼、Fischer→菲舍尔，无乱码无？。
2. **/en 同名排序** → /api/search ORDER BY 加 nationality 含「英」优先(拉丁名)。Smith→史密斯 居首。
3. **/places ASCII/变音符** → places 英文改包含匹配(命中"Tōkyō(Tokyo)"括注 ASCII)、is_crossref(见X)殿后、排序去前缀优先改最短优先。Tokyo→东京 居首。
4. **/en 数据噪声** → ORDER BY 加 NOTE_PENALTY(chinese 含标点=释义注殿后)。Lyon→莱昂/利翁 居首。
5. **/pinyin 缺字** → scripts/augment_given_pinyin.mjs 补 106 常用给名字。zhang wei→张伟 居首。
6. **/ko 罗马字** → search-ko 英文匹配去连字符+小写规范化。Seojun→서준/瑞俊 命中。
7. **纯音译质量引导（2026-06-05 第二批）** → /convert 顶部加黄底说明条："仅用于没有通用译名的普通名字"，举例 Dupont→迪蓬非杜邦，引导知名人物先用主搜/俄韩日专库（内链）。文案解决体验，非数据问题。

## 结论
核心动线（俄逐词、韩日汉字、地名正反查、转拼音、综合主搜常见名）**基本满足**。失分集中在**数据质量与排序**：德语音译崩、同名排序、ASCII/变音符、拼音高频字缺、数据噪声。这些是译者一眼能看出的硬伤，应优先于新功能。
